# Dependency constraints — pinned versions the test suite is known to pass
# against.  Companion to ``requirements.txt``, which remains the manifesto of
# direct dependencies with ``>=`` lower bounds.  This file pins those same
# top-level packages to a specific known-good version so CI and local
# installs are reproducible.
#
# Scope: only direct (top-level) dependencies are pinned here.  Transitive
# dependencies are deliberately NOT pinned — pip resolves them freshly
# within the window allowed by these top-level pins.  Rationale: this is a
# conservative middle-ground that protects against the main regression
# source (major version bumps of pandas / numpy / scikit-learn / xgboost /
# …) without the brittleness of a full transitive lock that would need to
# be regenerated per Python version.
#
# Usage — install with both files applied:
#
#     pip install -r requirements.txt -c constraints.txt
#
# The versions below were captured from a working local environment whose
# ``pytest -q`` run passes 156/156 tests.  To bump a pin: edit the value,
# run the suite locally, push the change in its own PR, confirm CI is
# green, merge.
